# Заметный донат + цветная главная иконка

**Дата:** 2026-06-12
**Статус:** одобрено пользователем (диалог 2026-06-12)
**Контекст:** первая волна обратной связи — пользователи спрашивают, как донатить.
Сейчас путь: главный экран → «О разработчике» (мелкая кнопка в футере) → «💛
Поддержать проект». Три клика, точку входа никто не находит.

## 1. Постоянная кнопка в футере главного экрана

Футер `#screen-main` (`extension/popup/popup.html`, блок `.app-footer`)
становится двухстрочным:

- **Строка 1** — полноширинная кнопка-ссылка `💛 Поддержать проект` →
  `https://yoomoney.ru/to/410011076392857`, `target="_blank"`,
  `rel="noopener noreferrer"`. Один клик вместо трёх.
- **Строка 2** — существующее «Сделано в Wildbots ↗ · О разработчике», без
  изменений.

Стиль кнопки — янтарный, по образцу существующего `.about-cta-heart`
(мягкий жёлтый фон `rgba(245,158,11,.12)`, янтарная рамка, `--amber-ink`),
но компактнее: высота ~34px, чтобы не раздувать попап. Новый класс —
`.footer-donate`.

Кнопка на экране «О разработчике» остаётся без изменений.

## 2. Баннер благодарности (деликатный режим)

Новый баннер `#donate-banner` на главном экране — первым элементом внутри
`.main-scroll` (над блоком «Сервисы»), чтобы при показе он был виден сразу,
но скроллился вместе с контентом. Стиль — в семействе `rkn-banner` /
`ai-free-banner`, но тёплый янтарный.

Содержимое: «Нравится ProxyPilot? Поддержи проект 💛» + кнопка-ссылка
**Поддержать** (тот же YooMoney URL) + крестик ✕.

### Логика показа

Триггер — «пользователь уже получил пользу», а не конкретный подбор
бесплатного прокси (иначе пользователи manual/own никогда не увидят баннер):

- При каждом открытии попапа, когда прокси **включён и активен**
  (`state.enabled === true` и активный источник настроен — те же условия,
  что `routeInitialScreen` использует для `hasManual/hasFree/hasOwn`),
  инкрементируется `donate.uses`. Не чаще одного раза за открытие попапа.
- Баннер показывается, когда выполняются все условия:
  - `donate.uses >= 3`;
  - `donate.dismissed !== true`;
  - `Date.now() - donate.lastShownAt >= 14 дней`.
- При показе баннера фиксируется `donate.lastShownAt = Date.now()`
  (один раз за popup-сессию).
- Крестик ✕ → `donate.dismissed = true` (навсегда), баннер скрывается.
- Клик по «Поддержать» в баннере → тоже `donate.dismissed = true`
  (человек уже отреагировал; футер-кнопка остаётся всегда).

## 3. Хранение

В `getDefaultState()` (`extension/lib/storage.js`):

```js
donate: { uses: 0, lastShownAt: 0, dismissed: false },
```

В `loadState()` — defensive backfill для обновляющихся пользователей
(по образцу `ownPool`): `if (!saved.donate) saved.donate = { ...defaults.donate };`

Тест в `tests/storage.test.js`: backfill `donate` при загрузке старого state.

## 4. Цветная главная иконка

Сейчас `manifest.json` указывает `icons/light/off-*.png` (серая «P») и в
`"icons"` (Chrome Web Store, chrome://extensions), и в `action.default_icon`.
Расширение везде выглядит серым.

- Создать брендовый набор `extension/icons/app-{16,32,48,128}.png` — копии
  `icons/light/routed-*.png` (фиолетово-синий градиент с бирюзовой точкой,
  как логотип в попапе). Копии, а не ссылки на routed: главная иконка не
  должна незаметно меняться при будущих правках state-иконок.
- `manifest.json` → `"icons"` и `action.default_icon` указывают на `app-*`.
- Тулбарная state-машина (`lib/icon.js`, `refreshTabIcon`) НЕ меняется:
  серая `off` при выключенном тумблере — информативный сигнал.

## 5. Версия и сборка

- Бамп версии: `0.11.13` → `0.12.0` в `extension/manifest.json` и
  `package.json`.
- `scripts/build.sh` подхватывает новые PNG автоматически (zip всего
  `extension/`).

## Вне скоупа (YAGNI)

- Альтернативные платёжки, телеметрия кликов, баннер на экране настроек.
- Изменение state-иконок тулбара (off/direct остаются серыми по смыслу).
