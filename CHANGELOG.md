# Changelog

Все заметные изменения проекта попадают сюда. Формат основан на
[Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
версионирование — [SemVer](https://semver.org/lang/ru/).

## [0.9.0] — 2026-06-10

### Добавлено
- **«Свой пул» — третий источник прокси** (Источник: Свой · Свой пул · Бесплатный).
  Вставь список своих прокси (по строке: `socks5://user:pass@host:port`,
  `http://host:port`, `host:port:user:pass`) — плагин подключит первый, а если он
  перестанет отвечать, **сам переключится на следующий** из твоего списка (мёртвый
  помечается на 30 мин, как в бесплатном пуле). Это решение проблемы дохлых
  бесплатных прокси: свой пул доверенных адресов с авто-ротацией.
  - Переиспользует механику ротации/пометки мёртвых; источник — твой список.
  - Выбор оптимистичный, без предварительной валидации: приватные прокси с auth
    проверяются по факту работы (`onAuthRequired` по `state.proxy`), а текущий
    можно проверить кнопкой «Проверить прокси». Кнопка «↻ Следующий» — вручную.

## [0.8.3] — 2026-06-09

### Исправлено
- **Бесплатный пул «все прокси мёртвые» + получасовой подбор.** Публичный список
  Proxifly сейчас состоит почти целиком из http-прокси без HTTPS (`https:false`,
  не туннелируют https-сайты) и transparent-socks (светят реальный IP). Плагин
  перебирал все ~366 кандидатов по 5 сек = до ~30 минут впустую.
  - `normalizePool` читает флаг `https`; SOCKS считаются HTTPS-способными (туннелируют любой TCP).
  - `filterPool` ставит HTTPS-способные прокси первыми.
  - `pickAndValidate` ограничен `MAX_VALIDATION_ATTEMPTS` (30) вместо «перебрать всё».
  - Человеческие сообщения вместо «проверено N из M»: если HTTPS-прокси в списке
    нет — честно «список почти бесполезен, укажи свой прокси».

## [0.8.2] — 2026-06-09

### Исправлено
- **Не переключался «Бесплатный пул».** Клик по источнику дизейблил обе вкладки
  и ждал ответа background, а подбор рабочего free-прокси (`pickAndValidate`)
  идёт десятки секунд — всё это время вкладки были заблокированы и без `:disabled`
  стиля выглядели как «не нажимается», а активная вкладка не переключалась до
  ответа. Теперь переключение **мгновенное и оптимистичное**: вкладка/блок
  меняются сразу, показывается «Подбираем рабочий прокси…», дизейбл убран —
  можно в любой момент вернуться на «Свой».

## [0.8.0] — 2026-06-09

Чистка легаси «Gemini Unblock» теперь, когда плагин — универсальный роутер на
39 сервисов.

### Исправлено
- **Google Labs ломал вход в Google.** Список «гугловых» ключей был захардкожен
  в трёх местах и разъехался (`pac.js`/`background.js` без `googleLabs`). Теперь
  единый источник — флаг `couplesGoogleAuth` в `presets.js`; включение Google Labs
  корректно подцепляет `accounts.google.com` (есть регресс-тест).
- **Предупреждение про free-прокси** показывалось только для Google-AI — теперь
  срабатывает для любого маршрутизируемого сервиса; текст нейтральный.
- **РКН при недоступном списке** больше не сбрасывает блокировки и не выключает
  всё подряд — состояние держится до следующей успешной проверки.

### Изменено
- **Нейтральные дефолты:** на свежей установке ничего не маршрутизируется (раньше
  Gemini + AI Studio + вход Google были включены). Новые пресеты при обновлении
  добавляются выключенными.
- **Самопроверка** «Проверить Gemini» → «Проверить сервис» (тестит первый
  включённый пресет), результат с названием сервиса.
- **Валидация free-прокси** бьётся в нейтральную точку, а не в Google.
- Флаг `isAi` → `couplesGoogleAuth`, `AI_PRESET_KEYS` → `GOOGLE_AUTH_PRESET_KEYS`.
- Дефолты `storage.js` выводятся из `PRESET_DEFINITIONS` (убрано дублирование
  ~40 доменных массивов). Иконочный роутинг использует общий `isHostRouted`
  из `pac.js` (убрана расходящаяся копия в `background.js`).

### Добавлено (UX на 39 сервисов)
- Поиск/фильтр по сервисам; включённые наверх категории; сворачиваемые категории
  со счётчиком; сводка «N включено» + «Сбросить»; внутренняя прокрутка грида.
  Убрано конфетти при добавлении домена.

### Гигиена
- Удалены легаси-артефакты (`extension-patch/`, дизайн-зип старого бренда, мёртвые
  `.gitkeep`), `*.zip` в `.gitignore`. Иконка Peacock 186 КБ → ~1 КБ. Тесты 92 → 98.

  (нужен резидентный прокси; бесплатный датацентр-пул для них не подойдёт).

### Исправлено
- CSS-коллизия: класс `.logo` использовался и для иконки в шапке, и для
  брендовых лого в гриде. Хедерная иконка переименована в `.logo-app`
  (`popup.css` + `popup.html`), брендовые лого остаются на `.preset-card .logo`.
- **Free-pool «список не обновляется».** Раньше `poolFetchedAt` ставился только
  при ручном выборе/ротации, фонового обновления не было вообще — подпись
  «Список обновлён N мин назад» зависала на времени последнего выбора (у
  пользователя — ~9.9 дней). Добавлен alarm `free-pool-refresh` каждые 5 минут
  (= TTL списка Proxifly): обновляет список Proxifly и таймстамп; если рабочий
  прокси не выбран — проактивно подбирает новый. Рабочий прокси НЕ перепроверяем
  активно (это гоняло бы весь трафик через free-прокси и моргало бы стримами
  каждые 5 мин) — мёртвый прокси ротируется реактивно при первой же ошибке
  запроса (как и раньше). Alarm'ы создаются через `ensureAlarm()` (не пересоздаём
  на каждый рестарт SW, иначе таймер короткого алярма сбрасывался бы и не срабатывал).

## [0.5.1] — 2026-05-28

### Исправлено
- Бесплатный пул больше не падает с `no working proxies found (tried 10)`
  на первой же попытке. Корень проблемы — Proxifly в `/all/data.json`
  держит в топе записи с `anonymity: transparent`, `country: ZZ`,
  `score: 1`. Такие прокси либо палят IP, либо мёртвые, либо мгновенно
  банятся Google.
- Cloudflare CDN trace, через который раньше проверяли живость, отвергает
  большинство free-IP с HTTP 403 — это L7-блаклист CF.

### Изменено
- Health-check переключён на `https://www.google.com/generate_204` —
  endpoint, спроектированный именно под connectivity checks (Android
  бьёт его миллиарды раз в сутки), без rate-limit, без тела для парсинга.
  Бонус: прокси, прошедший этот тест, с большой вероятностью пройдёт и
  `gemini.google.com`.
- `filterPool` отсекает `anonymity === 'transparent'` и `country === 'ZZ'`.
- Кандидаты сортируются по `score` DESC после shuffle — топ-качество
  вперёд, при равенстве остаётся случайность.
- Validate timeout: 3 s → 5 s (free-прокси часто отвечают 5–10 s).
- Попыток на проход: 10 → 15.
- `validateProxy` возвращает `{ ok, latencyMs, error }` — country теперь
  берётся напрямую из `candidate.country` Proxifly, парсера CF trace
  больше нет.

### Локализация
- Весь user-facing UI popup'а переведён на русский: лейблы, кнопки,
  плейсхолдеры, баннеры (AI-предупреждение, RKN), статусы (`Активно`,
  `Выключено`), сообщения тестов и автодетекта, тосты, ошибки добавления
  домена в `lib/domain.js`. `<html lang>` сменён на `ru`.
- Описание расширения (`manifest.json` → `description`) тоже на русском.
- По-английски оставлено: бренд `Gemini Unblock`, имена пресетов
  (`Gemini`, `AI Studio`, `ChatGPT`, …), названия протоколов
  (HTTP/HTTPS/SOCKS4/SOCKS5).

## [0.5.0] — 2026-05-28

Крупный релиз: добавлен альтернативный источник прокси —
[Proxifly](https://github.com/proxifly/free-proxy-list).

### Добавлено
- **Free pool** — режим где плагин сам подбирает рабочий прокси из
  общедоступного списка Proxifly. Включается в настройках pill-группой
  `Источник: Свой | Бесплатный пул`.
- Lazy-валидация: список тянется по триггеру (включение режима, кнопка
  `↻ Сменить`, ошибка прокси), кандидаты проверяются последовательно
  до первого живого.
- Авто-ротация при ошибках: `chrome.webRequest.onErrorOccurred` ловит
  `net::ERR_PROXY_*` / `ERR_SOCKS_*` / `ERR_TUNNEL_*` и автоматически
  переключается на следующего живого. Дебаунс 10 с + флаг
  `rotationInProgress` против гонок.
- Чёрный список стран `RU / BY / CN / IR` (бесполезны для Gemini),
  TTL мёртвых хостов 30 минут.
- Жёлтый баннер на главном экране при `Free pool + любой AI-пресет`:
  «Google может пометить аккаунт» — для типового кейса с Google-логином
  free-прокси настоятельно не рекомендуются.
- Сохранение `manualProxy` отдельно от активного `state.proxy`, чтобы
  переключение `Свой ↔ Бесплатный` не теряло введённые юзером значения.

### Изменено
- Схема стейта `v1 → v2`. Миграция:
  - `proxy → manualProxy` (копия),
  - `proxySource = 'manual'`,
  - инициализируется `freeProxy { selected, lastError, deadHosts, poolFetchedAt }`.
- Миграция идемпотентна, повторный `loadState` на v1-стейте всё ещё
  возвращает валидный v2.
- Кэш списка прокси трёхуровневый (memory → `chrome.storage` → сеть),
  TTL 5 минут — соответствует частоте обновления Proxifly.

### Технически
- Новый модуль `extension/lib/free-pool.js`: `fetchPool`, `filterPool`,
  `validateProxy`, `pickAndValidate`.
- Новые message handlers в background: `SWITCH_SOURCE`, `ROTATE_FREE`.
  `PERSIST_MANUAL` был запланирован но удалён — popup пишет
  `state.manualProxy` локально через `mirrorManual()`.
- Тесты: free-pool 17, storage migration 11, всего 83.
